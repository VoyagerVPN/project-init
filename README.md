# Project Tracker — шаблон трекинга для проектов

Система трекинга задач, журналирования прогресса и интеграции с Todoist для Cursor AI.

---

## Установка

### 1. Скопировать в проект

Скопируй содержимое этого репозитория в корень своего проекта:
```
.cursor/    → в корень
.tracker/   → в корень
scripts/    → в корень
backup.config.json → в корень
```

### 2. Настроить Todoist MCP

В Cursor: **Settings → MCP → Add Server** (или отредактировать `.cursor/mcp.json`):

```json
{
  "mcpServers": {
    "todoist": {
      "command": "npx",
      "args": ["-y", "@abhiz123/todoist-mcp-server"],
      "env": {
        "TODOIST_API_TOKEN": "ВСТАВЬ_СВОЙ_ТОКЕН"
      }
    }
  }
}
```

**Где взять токен:**
1. Открой [todoist.com/app/settings/integrations/developer](https://todoist.com/app/settings/integrations/developer)
2. Скопируй API Token
3. Вставь в `TODOIST_API_TOKEN`

> Без MCP сервера журнал и ошибки работают, но Todoist-интеграция (комментарии к задачам, "что на сегодня" и т.д.) будет недоступна.

### 3. Привязать проект Todoist

Скажи ИИ: **"настрой таск-трекинг"**

ИИ покажет список проектов из Todoist → выбери нужный → `.tracker/config.json` заполнится.

### 4. Готово

## Что внутри

```
.cursor/
├── rules/
│   ├── project.mdc       ← Карта всех команд (включён всегда)
│   ├── tracker.mdc       ← Todoist + журнал + pipeline
│   └── error-journal.mdc ← Логирование багов
├── commands/
│   ├── journal.md        ← Слеш-команда /journal
│   ├── report.md         ← Слеш-команда /report
│   └── backup.md         ← Слеш-команда /backup
.tracker/
├── config.json           ← Todoist project ID (заполняется при настройке)
├── journal.md            ← Журнал прогресса (по дням)
├── journals/template.md  ← Шаблон записи + примеры по категориям
└── errors.md             ← Реестр багов
scripts/backup.ps1        ← Скрипт архивации
backup.config.json        ← Исключения из архива
```

## Команды

| Скажи | Что произойдёт |
|---|---|
| **что на сегодня** | Задачи ПМа с дедлайном на сегодня |
| **давай [задачу]** | Контекст → план → работа → журнал + Todoist |
| **покажи задачу [X]** | Описание + подзадачи + комментарии |
| **что нового** | Что ПМ добавил/изменил |
| **все задачи** | Обзор проекта |
| **/journal** | Записать прогресс в журнал |
| **/report** | Отчёт за день (копировать в Telegram) |
| **залогируй ошибку** | Запись в errors.md |
| **backup** | Архивация проекта |

---

## Todoist-интеграция

ИИ **только пишет комментарии** к задачам ПМа. Не закрывает, не меняет приоритеты, не создаёт задачи.

ПМ ставит задачу → ты работаешь → ИИ пишет комментарий "что сделано" → ПМ видит прогресс.

---

## Журнал и ошибки

- **journal.md** — один файл, все записи по дням. Категории: 🚀 Фича, 🐛 Багфикс, 🎨 UI, 🔧 Рефакторинг, 📦 Инфра, 📝 Документ.
- **errors.md** — баги, которые не исправлены за сессию. С severity (🔴🟡⚪), гипотезами и шагами воспроизведения.
