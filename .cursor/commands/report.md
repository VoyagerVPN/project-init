# Отчёт / Журнал на сегодня (для ПМа)

**Триггеры:** `/report`, `отчёт`, `журнал на сегодня`, `что скинуть ПМу`, `отчёт за день`.

---

## Источник данных

Единственный журнал: `.tracker/journal.md`. Все записи по дням в секциях `## YYYY-MM-DD`.

---

## Действия

### Шаг 1: Дата и время
```powershell
Get-Date -Format "yyyy-MM-dd HH:mm"
```

### Шаг 2: Извлечь записи

Из `journal.md` — секция `## YYYY-MM-DD` (сегодня).

**Если записей за сегодня нет:**
```
📊 За сегодня записей в журнале нет.
Добавить запись? → /journal
```

### Шаг 3: Todoist — что закрыто

```
1. projectId из .tracker/config.json
2. find-completed-tasks(since="YYYY-MM-DD", until="YYYY-MM-DD", projectId=...)
3. Объединить с записями из journal.md
```

### Шаг 4: Сформировать отчёт

**Формат — готов для копирования в Telegram:**

```
📊 Отчёт за ДД.ММ.ГГ

1. ✅ Переключатель показателей (подробно/компактно)
2. 🐛 parseParamValue — whitelist форматов
3. 🎨 Стейт комментария, иконки, фидбэк кнопки "Обновить"
4. 🚀 childState/effectiveState — overspill статуса потоков

Баги: 0 открытых / 1 закрыт сегодня
```

**Правила формата:**
- Нумерованный список с иконками категорий.
- Каждый пункт — 1 строка, максимум 10 слов.
- Без технических деталей (файлы, модули — не для ПМа).
- Счётчик багов внизу.

### Шаг 5: Вывести

Показать готовый текст в чат. Пользователь копирует в Telegram/почту.

### Шаг 6: Опционально — сохранить

Спросить: "Сохранить в `.tracker/reports/YYYY-MM-DD.md`?"
Если да — создать файл (папку `reports/` создать если нет).

### Шаг 7: Опционально — Todoist

Спросить: "Отправить как комментарий к проекту в Todoist?"
Если да → `add-comments(projectId=..., content=отчёт)`.
