# Добавить в журнал / Прогресс

**Триггеры:** `/journal`, `/журнал`, `/прогресс`.

---

## Назначение

Зафиксировать выполненную работу в общем журнале. Использовать вместо ожидания фраз «всё» или «готово».

---

## Действия

### Шаг 1: Получить время
```powershell
Get-Date -Format "yyyy-MM-dd HH:mm"
```

### Шаг 2: Определить что сделано

Из контекста текущей сессии:
1. Проанализировать последние сообщения пользователя — что просил.
2. Проанализировать изменённые файлы — что фактически изменилось.
3. Определить категорию по типу работы:
   - `.tsx`, `.css`, стили → 🎨 UI
   - `.test.ts`, CI конфиги → 📦 Инфра
   - Исправление ошибки → 🐛 Багфикс
   - Новая функциональность → 🚀 Фича
   - Разбиение/чистка кода → 🔧 Рефакторинг
   - README, комментарии → 📝 Документ
4. Сформулировать **"Задача"** из запроса пользователя (что просил).
5. Сформулировать **"Решение"** из фактических изменений (что сделано).

### Шаг 3: Batch-режим (если за сессию сделано несколько вещей)

Если за сессию выполнено 2+ разных задач:
```
📝 Предлагаю 3 записи:

1. [🐛 Багфикс] parseParamValue — whitelist форматов (16:30)
2. [🎨 UI] Стейт комментария (17:00)
3. [🚀 Фича] Переключатель показателей (17:45)

Добавить все? / Изменить / Убрать
```

Если одна задача — стандартный формат (одна запись).

### Шаг 4: Подтверждение

Показать предлагаемую запись:
```
📝 Предлагаю запись:

### [🚀 Фича] Переключатель показателей (17:45)
**Задача:** Режимы «Подробно» / «Компактно».
**Решение:** Компакт — только карточки + badge, без показателей.
**Файлы:** `MonitorPage.tsx`, `MetricGroupCard.tsx`

Добавить так? Да / Изменить
```

### Шаг 5: Запись в journal.md

После одобрения:
1. Открыть `.tracker/journal.md`.
2. Найти секцию `## YYYY-MM-DD` (сегодня). Если нет — создать.
3. Добавить запись(и) в конец секции.

### Шаг 6: Todoist-интеграция

1. Прочитать `projectId` из `.tracker/config.json`.
   → Если пустой/нет → пропустить Todoist.
2. Извлечь ключевые слова из записи.
3. `find-tasks(searchText=ключевые слова, projectId=...)`.
   → **0 результатов:** "Задача Todoist не найдена. Записано в journal.md."
   → **1 результат:** "🔗 Привязать к «[название]»?" → если да, шаг 6.4.
   → **>1 результат:** показать список с номерами → "Какую привязать?"
4. `find-comments(taskId)` — проверить дубль (та же дата + тема).
5. Если дубля нет → `add-comments(taskId, content)` в формате:
```
📋 ДД.ММ.ГГ ЧЧ:ММ
[Что сделано, 1-2 предложения].
Файлы: file1.ts, file2.tsx
```

---

## Структура записи

```markdown
### [🚀 Фича] Название (ЧЧ:ММ)
**Задача:** что требовалось.
**Решение:** что сделано.
**Файлы:** `path/to/file.ts`
```

Полный шаблон и примеры: `.tracker/journals/template.md`.
