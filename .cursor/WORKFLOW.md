# Workflow: структура и инициализация

## Что где лежит

Каждый проект — своя копия. Общего репозитория между проектами нет.

| Путь | Содержимое | В архиве? |
|------|------------|-----------|
| `.brain/` | Brain MCP (project_id, server, token) — **не менять из правил** | ❌ |
| `.cursor/` | rules/, commands/, templates/, INIT_PROJECT_PROMPT | ❌ |
| `.tracker/` | Todoist config, journal.md (история), journals/template.md, errors.md | ❌ |
| `scripts/` | backup.ps1 и др. | ✅ |
| `backup.config.json` | exclude для архива | ✅ |
| Исходный код | agent/, monitor/, ... | ✅ |

**Итог архива:** код + scripts + backup.config.json. Правила и трекинг — нет.

## Инициализация нового проекта

```
┌─────────────────────────────────────────────────────────────┐
│  Источник                                                    │
│  • INIT_PROJECT_PROMPT.md (из любого настроенного проекта)   │
│  • scripts/ — откуда?                                        │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│  Промпт в чат нового проекта                                 │
│  "Инициализируй проект..."                                   │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│  ИИ создаёт                                                   │
│  • .cursor/rules/ (project, tracker, error-journal)          │
│  • .cursor/commands/backup.md                                │
│  • .tracker/ (config.json, journal.md, journals/template.md, errors.md) │
│  • backup.config.json                                        │
│  • scripts/ — ???                                            │
└─────────────────────────────────────────────────────────────┘
```

### Откуда брать scripts/

**Источник:** `.cursor/templates/backup.ps1` в мастер-проекте (videomax-new).

**При инициализации нового проекта:**

1. Открыть мастер-проект.
2. Скопировать INIT_PROJECT_PROMPT (блок ---BEGIN--- ... ---END---).
3. Скопировать `.cursor/templates/backup.ps1`.
4. Вставить промпт в чат нового проекта.
5. После создания правил/конфигов: вставить скрипт с текстом «Создай scripts/backup.ps1 с этим содержимым», либо скопировать вручную: `copy путь\к\мастеру\.cursor\templates\backup.ps1 путь\к\новому\scripts\`.

## Workflow: полный цикл

### 1. Первый раз (мастер-проект, например videomax-new)

- Вручную или через ранний init созданы: rules, commands, .tracker, scripts/backup.ps1, backup.config.json.
- .brain настроен через Brain MCP.
- Todoist привязан через «настрой таск-трекинг».
- INIT_PROJECT_PROMPT.md создан и содержит всё нужное для копирования в новый проект.

### 2. Новый проект

- Открыть мастер-проект → скопировать из INIT_PROJECT_PROMPT блок ---BEGIN--- ... ---END---.
- Открыть новый проект → вставить в чат.
- ИИ создаёт структуру. Если в промпте есть backup.ps1 — создаётся scripts/backup.ps1.
- «Настрой таск-трекинг» → выбрать проект Todoist.
- .brain — настроить отдельно (Brain ON + путь).

### 3. Архивирование

- `/backup` или «backup» → scripts/backup.ps1.
- Архив: `.temp/backup/project-yyyy-MM-dd-HH-mm.rar`.
- В архиве: код + scripts + backup.config.json (без .cursor, .tracker, .brain).

### 4. Восстановление из архива

- Распаковать архив → получить код + scripts + backup.config.json.
- Правила и трекинг нужно восстановить заново:
  - вставить INIT_PROJECT_PROMPT в чат, либо
  - скопировать .cursor/ и .tracker/ из другого бэкапа (если они где-то сохраняются отдельно).

---

## Рекомендации

1. **Мастер-проект** — videomax-new: правила, шаблоны, скрипты. Новые проекты инициализируются из него.
2. **Шаблоны** — `.cursor/templates/backup.ps1` хранит каноническую версию. scripts/backup.ps1 может отличаться (доработки); при обновлении шаблона — синхронизировать.
3. **Правила в архив** — если при восстановлении нужны правила, убрать `.cursor` из backup.config.json exclude.
