# Шаблон записи в журнал

Скопировать в `.tracker/journal.md` под нужной датой.

---

## Категории (с иконками)

| Иконка | Категория | Когда применять |
|--------|-----------|------------------|
| 🚀 | `[🚀 Фича]` | Новая функциональность, фича по ТЗ |
| 🐛 | `[🐛 Багфикс]` | Исправление бага, устранение сбоя |
| 🎨 | `[🎨 UI]` | Интерфейс, вёрстка, UX, визуальные правки |
| 🔧 | `[🔧 Рефакторинг]` | Улучшение кода без смены поведения |
| 📦 | `[📦 Инфра]` | Тесты, CI/CD, сборка, деплой, окружение |
| 📝 | `[📝 Документ]` | Правки в документации, комментарии в коде |

---

## Формат записи

```markdown
### [🚀 Фича] Название задачи
**Задача:** что требовалось сделать.
**Решение:** что сделано, ключевые моменты.
**Файлы:** `path/to/file.ts`, `other/file.tsx`
```

---

## Примеры по категориям

### [🚀 Фича] Переключатель показателей
**Задача:** Режимы «Подробно» / «Компактно» для карточек.
**Решение:** Компакт — только карточки и badge, без деталей.
**Файлы:** `MonitorPage.tsx`, `MetricGroupCard.tsx`

### [🐛 Багфикс] parseParamValue
**Задача:** Исправление парсинга значений.
**Решение:** Whitelist форматов, удалён generic numMatch.
**Файлы:** `metrics.ts`, `parseParamValue.ts`

### [🎨 UI] Стейт комментария
**Задача:** Более понятные комментарии к устройству.
**Решение:** Иконка активна (зелёная) / не активна (серая).
**Файлы:** `MainTab.tsx`

### [🔧 Рефакторинг] Вынос парсеров
**Задача:** Тестируемые модули.
**Решение:** Парсеры вынесены в отдельные файлы.
**Файлы:** `coolingParsers.ts`, `nvidiaSmiParsers.ts`

### [📦 Инфра] Покрытие тестами
**Задача:** Unit-тесты для критичных функций.
**Решение:** Agent 194, Monitor 53 теста.
**Файлы:** `agent/tests/unit/*`, `*.test.ts`

### [📝 Документ] Обновление README
**Задача:** Актуализировать инструкции.
**Решение:** Добавлен раздел по установке.
**Файлы:** `README.md`
